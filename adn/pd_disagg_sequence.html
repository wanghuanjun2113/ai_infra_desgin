核心流程说明

1. 初始化

Router 启动 ZMQ Service Discovery，动态发现 prefill/decode 实例。Mooncake 模式下额外查询每个 prefill 节点的 bootstrap server 获取 engine_id。

2. Stage 1: Prefill（Router → Prefill）

Router 通过负载均衡策略选择 prefill 实例，将请求改写为 max_tokens=1, stream=false，注入 kv_transfer_params.do_remote_decode=true。Prefill 执行完整 prefill 计算，KV Connector 将 KV Cache 推送到 Decode 实例，返回 kv_transfer_params。

3. KV Cache 传输（Prefill → Decode）

  • Mooncake (push): Prefill 通过 TransferEngine 主动推送 KV 到 Decode
  • NIXL (pull): Decode 收到 remote_host/port/block_ids 后主动拉取
  • NCCL: 通过 NCCL P2P 直连传输

4. Stage 2: Decode（Router → Decode）

Router 将 prefill 返回的 kv_transfer_params 附加到原始请求，发送给 decode 实例。Decode 加载远程 KV Cache 后跳过 prefill，直接逐 token 生成，流式返回给客户端。
